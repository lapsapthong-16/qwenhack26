# Repo Character IP

## Character Definition

The repo character is the default visual IP for README project illustrations.

By default, each image should include the repo character. The character is not a mascot, sticker, or cute decoration. It is an absurd worker seriously participating in the system.

## Appearance

- Small solid-black creature.
- White dot eyes.
- Thin legs, sometimes thin arms.
- Body can be a cylinder, black bean, black box, funnel, shadow, hole, or black block inside a machine.
- Slightly irregular hand-drawn outline.
- Blank, deadpan, calm, serious expression.

## Personality

- Very serious, but doing something slightly absurd.
- Feels like a low-profile system operator.
- Dry humor, not cute.
- Slightly awkward, not stupid.
- Looks truly responsible for one job inside a whiteboard sketch.

## Common Jobs

Make the repo character perform the core action:

- Carrying materials.
- Pulling lines to gather information sources.
- Getting stuck in a breakpoint.
- Operating a "judgment" lever inside a machine.
- Becoming a filtering funnel.
- Cutting open a "material fish".
- Stamping handoff copy.
- Holding a handoff path.
- Holding a warning sign near a pit.
- Reaching from a hole but failing to catch content.
- Carrying bricks, building a bridge, opening a door, sorting, or recording.

## Bans

- Do not make the repo character an overly cute mascot.
- Do not make it a children's cartoon character.
- Do not give it complex clothes, meme expressions, or shiny eyes.
- Do not leave it standing in the corner watching.
- Do not let it overpower the structure.
- Do not make it too commercial, rounded, or polished.

## Standard

If the core metaphor still works perfectly after removing the repo character, the character is too decorative. Rewrite the prompt so the repo character becomes the action subject.
